# Архитектура приложения 

Код в приложении распределен на слои согласно парадишме MVP:
  - Model слой данных , отвечает за хранение и изменение данных.
  - View слой представления, работает с отображением данных.
  - Presenter является связующим звеном между Model и View и наоборот.

# Базовый Код

### Класс Api

Содержит логику отправки запросов. В конструктор передаётся базовый адрес сервера и опциональный обьект с заголовками.

Методы: 
- get выполняет GET запрос на переданный адрес и возвращает промис, которым ответил сервер.
- post принимает обьект с данными и отправляет эти данные на переданный адрес, метод запроса может быть изменён, изменением метода при вызове.

### Класс EventEmitter

Брокер событий позволяет подписываться на события, инициировать их и управлять подписками. 

Методы:
 - on установка обработчика на определённое событие
 - off снятие установленного обработчика события
 - emit инициирует событие с переданными данными
 - оnAll подписывает на все события
 - offAll удаляет зарегистрированные обработчики для всех событий
 - trigger возвращает возвращает функцию, при вызове которой выполняется требуемое событие.


## Model cлой данных 

# Класс Model

Организует хранение и обработку данных приложения.
В полях хранятся следующие данные:
 -- totalProducts: number - суммарное количество товаров в галерее.
 -- gallery: IProduct[] - массив из обьектов карточек.
 -- basket: Map<string, number> - товары добавленные в корзину.
 -- order: IOrder - объект с данными, необходимыми для оформления заказа.
 -- events: IEvents - экземпляр EventEmitter для инициализации событий при изменении данных.    
 
Методы для работы с данными:
 -- getProduct(productId: string): IProduct - возврат карточки по id.
 -- addProductBasket(productId: string): void - добавление товара в корзину.
 -- removeProductBasket(productId: string): void - удаление товара из корзины
 -- геттеры и сеттеры для получения и сохранения данных.

## View слой представления.

Классы работают с отображением данных в контейнере (DOM-элемент).
 
 # Класс Card
Работает с отображением карточки, создавая в карточке данные названия, описания, категории, изображения, цены.
В классе установливается слушатель на кнопку добавления товара в корзину. Поля класса содержат элементы разметки элементов карточки.  Конструктор, кроме темплейта принимает экземпляр EventEmitter для инициализации событий.

 -- setData(cardData: IProduct): void - заполняет атрибуты элементов
карточки данными.
 -- render(): HTMLElement - метод возвращает заполненную карточку с установленным слушателем.
  -- геттер id возвращает уникальный id карточки.

#  Класс Modal
Реализует модальное окно. Отвечает за методы close и open для взаимодействия с окном, работает со слушателями на клавиатуру, для закрытия модального окна по клику на оверлей, крестик.
 -- constructor(selector: string, events: IEvents) конструктор принимает селектор, по которому на странице будет отображено модальное окно и класс EventEmitter для инициализации событий.

Поля класса 
 -- modal: HTMLElement - модальное окно.
 -- events: IEvents - брокер событий. 

# Класс CardsContainer
Отвечает за отображение карточек на главной странице. Имеет метод addCard(cardElement: HTMLElement) для добавления карточек на страницу. В конструктор принимет контейнер, для арзмещения карточек.

# Класс Order
 Содержит информацию о заказе, в полях для хранения данных. Принимает в констуктор экземпляр EventEmitter для инициализации событий при нажатии на кнопку.
 - геттеры и сеттеры для постепенного заполнения данных о заказе.

## Presenter связующий слой.
Принимает в конструктор экземпляр класса Api и реализует методы для взаимодейтсвия Model и View.
В Index.ts сначала создаются экземпляры всех необходимых классов, далее обработка событий.
События изменения данных, создаются классами моделями данных:
 -- basket:changed - изменение корзины.
 -- order:changed - изменение заказа.

 События при взаимодейтсвии пользователя с интерфейсом, создаются классами отображение.
    -- card:select - выбор карточки для отображения.
    -- basket:open - открытие окна с корзиной. 
    -- delivery:open - открытие окна для заполения данных о доставке.
    -- client:open - открытие окна для заполнения данных о клиенте.
    -- success:open - открытие окна с информацией об успешном оформлении заказа.
    -- basket:add - добавление товара в корзину.
    -- basket:remove - удаление товара из корзины.
    -- payment:input - изменение данных об оплате.
    -- address:input - измение данных с адресом.
    -- email:input - изменение адреса почты.
    -- telephone:input - изменение номера телефона.
    -- basket:submit - сохранение корзины.
    -- delivery:submit - сохранение данных доставки.
    -- client:submit - сохранение данных клиента.
